激活个人隐私数据保护特性

  • 操作场景
  • 必备事项
  • 操作步骤

操作场景

为保证个人隐私数据的安全性，建议运营商遵从所在国家的相关法律开启假名化/匿名化功能，对跟踪、日志、CHR中使用的用户个人标识进行保护。

适用于SGSN、MME、GGSN、SGW_C、PGW_C、AMF、SMF、NRF、NSSF。

必备事项

前提条件

请仔细阅读WSFD-010305 个人隐私数据保护特性概述。

数据

该操作无需准备数据。

操作步骤

  • 配置跟踪匿名化功能。

    Option

    Description

    通过网管

    详细操作步骤请参见U2020/MAE产品文档的“假名化策略管理”。

    须知：
    • U2020/MAE上假名化/匿名化开关默认开启，建议运营商遵从所在国家的相关法律。
    • 为更好地保护用户个人隐私，请定期更新并妥善保存HMAC密钥，避免被他人获取。密钥的长度建议符合安全要求，提升安全性。

    通过OM Portal
    • 当UNC未连接U2020/MAE之前，根据OM Portal跟踪“下载”界面的“匿名化”选项状态判断是否做匿名化处理。
    • 当UNC连接U2020/MAE之后，同步保存U2020/MAE上配置的假名化/匿名化功能状态及HMAC密钥信息：
      • 如果U2020/MAE上配置匿名化功能为开启，则强制做匿名化处理；
      • 如果U2020/MAE上配置匿名化功能为关闭，则根据OM Portal跟踪“下载”界面的“匿名化”选项状态判断是否做匿名化处理。
  • 配置日志匿名化功能。

    通过MML命令SET COMMONSOFTPARAOFBIT设置软件参数DWORD1 BIT1为0。

    通过MML命令SET FWSOFTPARABITS设置软件参数DWORD1601 BIT1为0。

    SET COMMONSOFTPARAOFBIT: DT=DwCom, DWORDCOMNUM=1, VALUE=0, POSITION=1;
    SET FWSOFTPARABITS: PARATYPE=DWORD, DWORDNUM=1601, VALUE=VALUE_0, POSITION=1;

  • 配置CHR假名化功能。

    Option

    Description

    通过网管

    详细操作步骤请参见U2020/MAE产品文档的“假名化策略管理”。

    须知：
    • U2020/MAE上假名化/匿名化开关默认开启，建议运营商遵从所在国家的相关法律。
    • 为更好地保护用户个人隐私，请定期更新并妥善保存HMAC密钥，避免被他人获取。密钥的长度建议符合安全要求，提升安全性。

    通过MML

    执行MML命令SET PSEUDONYPOLICY开启本地CHR假名化功能，并配置假名化口令。假名化口令为密码类型，长度必须为偶数（最小16位，最大30位），且只能输入0-9、A-F、a-f。

    SET PSEUDONYPOLICY: LOCALPOLICY=ON, KEY="*****";
    • 当UNC未连接U2020/MAE，或已连接U2020/MAE但U2020/MAE上未进行假名化/匿名化功能设置时，根据UNC上MML命令配置判断是否做假名化处理。
    • 当UNC已连接U2020/MAE，且U2020/MAE上已有假名化/匿名化功能配置时，根据U2020/MAE上配置的假名化/匿名化功能状态判断是否做假名化处理。
