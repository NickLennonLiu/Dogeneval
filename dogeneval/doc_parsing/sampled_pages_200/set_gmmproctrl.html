  • 使用实例

命令功能

适用网元：SGSN

在用户接入Gb模式时，UNC可通过此命令控制MM流程进行特殊处理以及特殊场景下指定的原因值下发，以满足运营商实现对现网设备兼容性特殊控制的需求。

注意事项

  • 系统初次运行时，会执行系统初始设置值。
  • 此命令执行后立即生效。
  • 配置下发的原因值可能会对终端行为产生影响，在配置前评估影响。关于不同原因值的含义及对终端行为的影响请参见协议3GPP TS 24.008。

本地用户权限

manage-ug；system-ug

网管用户权限

G_1，管理员级别命令组；G_2，操作员级别命令组

参数说明

参数标识

参数名称

参数说明

PROT

流程类型

可选必选说明：必选参数

参数含义：该参数用于指定流程类型，要根据场景来配置相应的流程。

数据来源：整网规划

取值范围：
  • “ATTACH(附着流程)”
  • “INTRA_RAU(USN内路由区域更新流程)”
  • “INTER_RAU(USN间路由区域更新流程)”
  • “UPDATE_LOCATION(位置更新流程)”
  • “AIR(获取鉴权集流程)”
  • “AUTHENTICATION(鉴权流程)”
  • “CHECK_IMEI(检查IMEI流程)”
  • “PAGING(寻呼流程)”
  • “CANCEL_LOCATION(位置取消流程)”

系统初始设置值：“ATTACH(附着流程)”

ULRIMSIGTERR

ULR拒绝原因值（IMSIGT错误）

可选必选说明：可选参数

参数含义：该参数用于设置因IMSIGT错误而导致ULR流程失败时，下发给终端的原因值。

前提条件：该参数在“流程类型”参数配置为“UPDATE_LOCATION(位置更新流程)”后生效。

数据来源：整网规划

取值范围：0～111

系统初始设置值：0

配置原则：
  • 参数设置为0，表示不使用特殊指定原因值，发生该异常时系统将下发#111（Protocol error unspecified）原因值。
  • 原因值为1表示非协议定义原因值，不建议配置。
  • 参数修改为非0值，可能会改变此参数所控制的异常场景中下发给终端的拒绝原因值，从而导致相应拒绝类话统指标的变化。

ULRHLRHSSREJ

ULR拒绝原因值（HLR/HSS拒绝）

可选必选说明：可选参数

参数含义：该参数用于设置因HLR/HSS设备拒绝而导致ULR流程失败时，下发给终端的原因值。HLR/HSS拒绝原因值有多种，SGSN根据映射规则将HLR/HSS发送的Update-Location-Answer消息中携带的原因值转换为在Attach/RAU Reject消息中的GMM Cause发送给UE。

前提条件：该参数在“流程类型”参数配置为“UPDATE_LOCATION(位置更新流程)”后生效。

数据来源：整网规划

取值范围：0～127

系统初始设置值：0

配置原则：
  • 参数取值为非0值，则它必须为ADD CAUSEGRP中已经存在的CAUSEGRPID（原因值组标识）值，并需要在ADD CAUSEMAP中根据HSS不同的拒绝原因值配置具体的原因值映射规则。这样就会改变此参数所控制的异常场景中下发给终端的原因值，从而导致UE行为的变化。

  • 参数取值为0表示不使用特殊指定原因值。UNC系统使用如下原因值映射：

    Gr接口原因值映射规则如下：

    • #1(Unknown Subscriber) —> #7(GPRS services not allowed)；
    • #8(Roaming Not Allowed) —> #14(GPRS services not allowed in this plmn)；
    • 其它 —> #111(Protocol error，unspecified)。

    S6d接口原因值映射规则如下：

    • #5001(DIAMETER_ERROR_USER_UNKNOWN) —> #15 (No suitable cells in location area)；
    • #5420(DIAMETER_ERROR_UNKNOWN_EPS_SUBSCRIPTION)，without Error Diagnostic，or with Error Diagnostic of GPRS_DATA_SUBSCRIBED —> #15 (No suitable cells in location area)；
    • #5420(DIAMETER_ERROR_UNKNOWN_EPS_SUBSCRIPTION)，with Error Diagnostic of NO_GPRS_DATA_SUBSCRIBED —> #7(GPRS services not allowed)；
    • #5004(DIAMETER_ERROR_ROAMING_NOT_ALLOWED)，without Error Diagnostic，or with Error Diagnostic of ODB_VPLMN_APN —> #11(PLMN not allowed)；
    • #5004(DIAMETER_ERROR_ROAMING_NOT_ALLOWED)，with Error Diagnostic of ODB_ALL_APN or ODB_HPLMN_APN —> #14(GPRS services not allowed in this PLMN)；
    • #5421(DIAMETER_ERROR_RAT_NOT_ALLOWED) —> #15 (No suitable cells in location area)；
    • #4181(DIAMETER_AUTHENTICATION_DATA_UNAVAILABLE) —> #7(GPRS services not allowed)；
    • 其他 —> #17 (Network failure)。

ULRREJIMSIUNKNOWN

ULR拒绝原因值（Diagnose为IMSI Unknown）

可选必选说明：可选参数

参数含义：该参数仅用于设置HLR/HSS返回原因值“Unknown Subscriber”，并且Diagnose为“IMSI Unknown”导致ULR（UPDATE_LOCATION）失败时，下发给终端的原因值。此配置的生效优先级高于ULRHLRHSSREJ（ULR拒绝原因值（HLR/HSS拒绝））的配置。

前提条件：该参数在“流程类型”参数配置为“UPDATE_LOCATION（位置更新流程）”后生效。

数据来源：整网规划

取值范围：0～111

系统初始设置值：0

配置原则：
  • 参数取值为0，表示不使用特殊指定原因值，如果发生该场景的失败，将由参数ULRHLRHSSREJ配置的原因值控制发送给终端，如果ULRHLRHSSREJ也未配置指定的原因值，将下发#7（GPRS services not allowed）原因值。
  • 原因值为1表示非协议定义原因值，不建议配置。
  • 非0值，则此失败场景，将使用本参数设置的原因值发送给终端。另外，使用非0值，发送给终端的原因值会发生变化，导致相应拒绝类话统指标的变化。

DETACHCAUSE_CLR

分离请求原因值

可选必选说明：可选参数

参数含义：HLR/HSS由于欠费等原因注销某用户，发送Cancel Location（原因值为Subscription Withdrawn）给SGSN。SGSN会针对该用户发起分离流程，发送Detach Request消息给UE，本参数控制是否在Detach Request消息中携带Cause信元，以及携带的Cause信元取值。

前提条件：该参数在“流程类型”参数配置为“CANCEL_LOCATION(位置取消流程)”后生效。

数据来源：整网规划

取值范围：0~111

系统初始设置值：7

配置原则：
  • 参数取值为0，给UE发送的Detach Request消息不携带Cause信元。
  • 参数取值1~111，给UE发送的Detach Request消息携带Cause信元，信元的取值为本参数取值。
  • 推荐设置原因值为7 GPRS services not allowed。
说明：

不同的原因值可能会导致UE的行为发生变化。推荐值7表示GPRS services not allowed，UE收到该原因值的Detach request，可能需要UE开关机才可以重新尝试附着网络，如果修改为非推荐值，例如111，未签约的UE将一直尝试重新附着，会增加核心网设备的负荷。

使用实例

当下发的缺省原因值导致位置更新流程异常时，设置“PROT（流程类型）”为“UPDATE_LOCATION（位置更新流程）”，“ULR拒绝原因值（IMSIGT错误）”为“2”，“ULR拒绝原因值（SCCPGT错误）”为“2”，“ULR拒绝原因值（链路异常）”为“10”，“ULR拒绝原因值（HLR/HSS超时）”为“20”的Gb模式移动性管理流程控制参数：

SET GMMPROCTRL: PROT=UPDATE_LOCATION, ULRIMSIGTERR=2, ULRSCCPGTERR=2, ULRLINKERR=10, ULRHLRHSSTMOUT=20;

当需要修改参数“ ULR拒绝原因值（HLR/HSS拒绝）”或“ AIR拒绝原因值（HLR/HSS拒绝）”为非“0”值时，则需要配置对应的原因值映射规则，以“ ULR拒绝原因值（HLR/HSS拒绝）”为例，下面配置“原因值组标识”为“1”的映射规则，将原始原因值“1”转换成“目标原因值”为“7”，配置顺序如下：

ADD CAUSEGRP: CAUSEGRPID=1, CAUSEGRPNM="GB-ULR-HLRHSS-REJ";

ADD CAUSEMAP: CAUSEGRPID=1, CAUSERANGE=SPECIAL, BGCAUSE=1, EDCAUSE=1, TCAUSE=7;

SET GMMPROCTRL: PROT=UPDATE_LOCATION, ULRHLRHSSREJ=1;

用户输入的ULRHLRHSSREJ或AIRHLRHSSREJ必须在CAUSEGRP（原因值映射组）中存在，否则执行失败。
