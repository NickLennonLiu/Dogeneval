设置HTTP消息日志参数（SET HTTPMSGLOGPARA）

  • 命令功能
  • 注意事项
  • 参数说明
  • 使用实例

命令功能

该命令用于设置HTTP消息日志的参数，该参数设置后整系统生效。

注意事项

  • 该命令执行后立即生效。
  • 系统部署完成后，已经存在初始记录，参数的初始记录值如下表：
TCPMSGSW STOPCPUTHOLD RESCPUTHOLD

OFF

75

65

操作用户权限

G_1，管理员级别命令组；G_2，操作员级别命令组

参数说明

参数标识 参数名称 参数说明

TCPMSGSW

TCP消息日志开关

可选必选说明：可选参数

参数含义：该参数用于配置是否开启TCP消息日志功能。开启该功能后，会记录TCP的SYN、FIN、RST消息及其后续部分消息的报文头内容。

数据来源：本端规划

取值范围：

  • “ON（打开）”：打开
  • “OFF（关闭）”：关闭

默认值：无。执行命令并不输入该参数时，系统默认为参数的初始设置值。

配置原则：无

STOPCPUTHOLD

停止记录的CPU阈值

可选必选说明：可选参数

参数含义：该参数用于配置停止记录消息日志的CPU占用率阈值。当CPU占用率大于或等于该阈值时停止记录消息日志。

数据来源：本端规划

取值范围：整数类型，取值范围是1~100，单位是百分比。

默认值：无。执行命令并不输入该参数时，系统默认为参数的初始设置值。

配置原则：

该参数的配置值必须大于参数“恢复记录的CPU阈值”的配置值。

RESCPUTHOLD

恢复记录的CPU阈值

可选必选说明：可选参数

参数含义：该参数用于配置恢复记录消息日志的CPU占用率阈值。当CPU占用率小于或等于该阈值时恢复记录消息日志。

数据来源：本端规划

取值范围：整数类型，取值范围是1~100，单位是百分比。

默认值：无。执行命令并不输入该参数时，系统默认为参数的初始设置值。

配置原则：

该参数的配置值必须小于参数“停止记录的CPU阈值”的配置值。

使用实例

如果需要打开HTTP消息日志开关，可以执行如下命令：

SET HTTPMSGLOGPARA: TCPMSGSW=ON;
