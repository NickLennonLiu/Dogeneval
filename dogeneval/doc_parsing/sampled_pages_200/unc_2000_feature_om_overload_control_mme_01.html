调测系统过载控制功能（适用于SGSN、MME）

  • 操作场景
  • 必备事项
  • 操作步骤

操作场景

支持当网络中出现CPU占用率达到了系统流控的阈值时，系统开始对相应进程进行流控。

适用于SGSN、MME。

必备事项

前提条件

  • 请仔细阅读系统过载控制功能特性概述（适用于SGSN、MME）。
  • 完成激活系统过载控制功能。

数据

该操作无需准备数据。

工具

该操作无需准备其他工具。

操作步骤

  1. 进入“MML命令行-UNC”窗口。
  2. 通过如下命令查看各流控参数与规划是否一致。

    Option

    Description

    Inter-MME流控参数

    LST INTERMMEFC

    LST INTERMMEFIXFC

    如设置了Inter-MME Service Request及Attach的流控相关软参BYTE_EX_B339和BYTE_EX_B340，则还需要执行如下命令查询软参设置情况：

    LST SOFTPARAOFBIT:DT=BYTE_EX;

    S1寻呼流控参数

    LST S1PAGINGFC

    如开启PAE寻呼反压流控功能，则还需要检查如下参数：

    • LST PAECHECKPARA
    • LST PAECHECKTHD
    • LST REVERSEFCPARA

    系统流控参数

    LST FLOWCTRLPARA:PROCTYPE=进程类型;

    流控队列参数

    LST FLOWCTRLQUE:APPTYPE=业务类型;

    S1模式基于TALIST寻呼消息不重发TAC区间

    LST TALISTPAGINGCFG

    预期结果：参数取值和规划一致，执行步骤 3。否则，参考激活系统过载控制功能（适用于SGSN、MME）调整配置。

  3. 触发各场景的消息过载，查看是否上报对应流控告警。

    Option

    Description

    Inter-MME流控

    ALM-80671 Inter-MME接入流控

    S1寻呼流控

    ALM-80649 S1模式寻呼流控

    自保流控

    ALM-80667 系统流控启动

    队列流控

    ALM-80667 系统流控启动
