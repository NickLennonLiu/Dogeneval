操作场景

  • 运营商合并和重组时，为了兼容现网中使用相同资源的多个APN，可将某APN的业务映射到另一APN上。

  • 网络改建时新规划了APN，为了不影响原规划APN的使用，只需将原规划的APN映射到新规划APN上即可。

适用于GGSN、PGW-C、SMF。

前提条件

  • 请仔细阅读WSFD-106203 别名APN特性概述（适应于GGSN/PGW-C/SMF）。

  • 完成加载license（如果有需求，请联系华为技术支持工程师处理）。

操作步骤

  1. 进入“MML命令行-UNC”窗口。
  2. 打开本特性的License配置开关。

    SET LICENSESWITCH:

  3. 配置真实APN。

    ADD APN

  4. 配置别名APN和转换APN。

    ADD APNALIAS

    1. 一个APN下最多可以配置500个APN别名。

    2. 系统最多可以配置1000个APN别名。

    3. ADD APNALIAS中的别名APN不能与ADD APN中已配置的APN相同。
    4. ADD APNALIAS中的转换APN必须已经在ADD APN中配置。

    5. ADD APNALIAS中，别名APN与转换APN为多对一的关系，但是不同的转换APN下不能配置相同的别名APN。对于5G用户，在不同切片下，不同转换APN可以配置相同别名APN。

  5. 删除别名APN配置。

    RMV APNALIAS

    当某个别名APN下还存在用户或会话时，该别名APN不允许直接删除，执行RMV APNALIASUNC会拒绝该请求。如果想删除此类别名APN，需要先锁定此别名APN，再批量去激活该别名APN下用户，最后删除别名APN。

任务示例

任务描述

配置别名APN为"apn1"和"apn2"，转换后的真实APN为“apn5”。

脚本

  1. 开启别名APN功能License。

    SET LICENSESWITCH:LICITEM="LKV2AAPN01",SWITCH=ENABLE;
  2. 别名APN名称为“apn1”和“apn2”，配置真实APN为 “apn5”。
    ADD APNALIAS:SUBRANGE=ALL_USER,ALIASAPN="apn1",CONVERTAPN="apn5";
    ADD APNALIAS:SUBRANGE=ALL_USER,ALIASAPN="apn2",CONVERTAPN="apn5";
