操作步骤

    在相关网元均完成本特性的配置后，可以采用以下步骤检查特性工作是否正常：

    1. 进入“MML命令行-UNC”窗口。
    2. 在UNC上创建用户跟踪任务，假设终端的IMSI为“460036901000001”。

    3. 设置重发寻呼的最大次数“N3413”为“1”。

      SET EMM: N3413=1;

    4. 设置精准寻呼动作为邻接eNodeB。

      ADD S1PAGINGRULE: GRPTYPE=ALL_USER, SVRTYPE=S11_DOWNLINK, APNIND=ALL_APN, ACTGRP=NEIGH_ENODEB-1, DESC="data";

    5. 设置“eNodeB覆盖增强检查开关”为“OFF(关闭)”。

      SET M2MCTRL: CHK_ENB_CE_SW=OFF;

    6. 终端在覆盖等级为2的小区下开机通过eNodeB1附着到网络（eNodeB1支持在UE Context Release Complete消息中携带寻呼辅助信息）。
    7. 终端移出eNodeB1到覆盖等级为3的eNodeB2下的小区。
    8. 执行下述命令，观察终端的MM上下文。

      DSP COMMMCTX: QUERYOPT=BYIMSI, IMSI="460036901000001";

      %%DSP COMMMCTX: QUERYOPT=BYIMSI, IMSI="460036901000001";%%
RETCODE = 0  操作成功
