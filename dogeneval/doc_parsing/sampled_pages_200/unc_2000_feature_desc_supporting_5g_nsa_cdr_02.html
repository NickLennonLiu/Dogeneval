适用NF

SGW-C、PGW-C

定义

NSA组网场景下，UNC支持将E-UTRAN上报的5G用户流量单独生成话单，上报给计费网关CG，并进一步将话单传递给计费中心BS。

客户价值

受益方

受益描述

运营商

支持5G用户流量离线计费，在4G/5G融合场景下，区分5G用户单独计费。

用户

记录5G用户使用数据业务的详细情况，便于核减消费情况。

应用场景

该特性适用于NSA Option3/3a/3x组网以及NSA Option3/3a/3x组网与2/3/4G组网共存场景。

应用限制

  • SGW-C/PGW-C支持最多与32个CG对接。

  • CG支持的话单版本字段需要与SGW-C/PGW-C的话单版本字段支持能力一致。

原理概述

NSA组网场景下，终端能同时建立到eNodeB和gNodeB的无线承载，并且在各自承载上能独立进行上下行的用户数据传输。EPC侧无法区分是4G流量还是5G流量。当需要对5G流量单独计费时，需由gNodeB进行流量统计，并通过接口发给eNodeB。随后eNodeB以承载的粒度向MME上报5G计费流量统计，并由MME转发给SGW-C/PGW-C。SGW-C/PGW-C根据MME上报的5G计费流量统计，生成话单上报到CG。NSA组网场景下5G数据的计费如图1所示。
图1 NSA组网场景下（以Option3x为例）5G数据的计费示意图

1.gNodeB进行5G流量统计。

2.gNodeB通过接口发给eNodeB。

3.eNodeB将消息转发至MME，消息中携带5G计费流量统计报告（Secondary RAT Usage Report List，参见3GPP TS 36413）。eNodeB可以在如下消息中携带5G计费流量统计报告：
  • E-RAB MODIFY RESPONSE
  • E-RAB RELEASE RESPONSE
  • E-RAB RELEASE INDICATION
  • E-RAB MODIFICATION INDICATION
  • UE CONTEXT RELEASE REQUEST
  • UE CONTEXT RELEASE COMPLETE
  • UE CONTEXT SUSPEND REQUEST
  • SECONDARY RAT REPORT
4.MME从eNodeB发送来的S1接口消息中解析到Secondary RAT Usage Report List信元后，根据当前的流程状态上报5G计费流量到SGW-C/PGW-C。MME可以在如下消息中携带Secondary RAT Usage Data Report信元：
  • Create Session Request
  • Modify Bearer Request
  • Delete Session Request
  • Delete Bearer Response
  • Delete Bearer Command
  • Release Access Bearers Request
  • Change Notification Request

5.SGW-C/PGW-C生成话单并上报到CG。SGW-C/PGW-C需配置ADD CDRFIELDTEMP命令中的“RAN-SecondaryRAT-Usage-Report”参数，使话单支持携带RAN-SecondaryRAT-Usage-Report字段。

  • 5G数据流量需要在SGW-CDR/PGW-CDR话单中区分上下行。
  • SGW-C/PGW-C支持R8、R9和R10版本话单。
  • SGW-C/PGW-C话单支持携带5G用户扩展QoS字段。

计费与话单

话单中携带RAN-SecondaryRAT-Usage-Report字段。
