告警解释

适用NF

AMF

  • 告警触发机制

    在“MML命令行-UNC”窗口执行命令MOD AMFINFO更新AMF配置，AMF通知NG-RAN，当未收到NG-RAN的响应，或者收到NG-RAN响应配置更新失败时，产生此告警。

    判断响应时长是通过ADD NGAPPARA中“AMF Configuration Update消息超时定时器”设置。

  • 告警恢复机制

    在“MML命令行-UNC”窗口执行命令MOD AMFINFO更新AMF配置，AMF重新通知NG-RAN，NG-RAN响应成功时，或者NG-RAN重新建链后，该告警恢复。

对系统的影响

"AMF名称"、"相对容量"、"GUAMI列表"、"PLMN列表"、"TNL偶联列表"这些配置无法发给NG-RAN，会导致NG-RAN端保存的AMF配置信息跟AMF的实际配置不一致，对系统可能会造成如下影响：

  • AMF名称更新失败时，可能会导致NG-RAN查找对应的AMF失败，影响用户接入。
  • 相对容量配置更新失败时，可能会导致Pool内各个AMF之间的用户均衡性分布不符合预期的容量配置。
  • GUAMI列表更新失败时，可能会导致NG-RAN根据5G-GUTI为用户选择AMF时，选择错误。
  • PLMN列表更新失败时，可能会导致NG-RAN为用户选择的AMF可能不支持该PLMN，导致用户接入失败。
  • TNL偶联列表更新失败时，可能会导致多偶联建立失败。

可能原因

  • 传输网络故障。
  • NG-RAN异常。

处理步骤

    排查传输网络故障

    1. 在“MML命令行-UNC”窗口执行命令DSP NGAPLINK查询NGAP链路状态，输入参数为移动国家码、移动网号、NG-RAN节点类型和NG-RAN节点标识。

      • 正常 => 步骤2
      • 故障 => 步骤3
      • 接入中 => 2
      • 查询不到响应记录 => 步骤3

    2. 等待1分钟后检查本告警是否已清除。

      • 是 => 处理完毕。
      • 否 => 步骤4

    3. 在“MML命令行-UNC”窗口执行命令PING，输入对端NG-RAN地址，和相应的本地IP地址，检查IP层链路是否连通。本地IP地址可咨询NG-RAN维护人员。

      • IP层链路通 => 步骤6
      • IP层链路不通 => 针对“IP链路故障”进行故障定位。

    排查是否NG-RAN异常

    1. 建立N2接口跟踪，输入NG-RAN的标识为告警参数中的NG-RAN节点标识，重新修改AMF配置以触发AMF配置更新。观查N2接口跟踪是否有NG-RAN对AMF配置更新请求的响应。

      • NG-RAN响应配置更新失败或无响应 => 步骤6
      • NG-RAN配置更新成功 => 步骤5

    2. 等待1分钟后检查本告警是否已清除。

      • 是 => 处理完毕。
      • 否 => 步骤10

    3. 与NG-RAN维护人员确认：NG-RAN侧是否进行了关闭链路或关闭端口的操作。

      • 是 => 步骤7
      • 否 => 步骤8

    4. 在NG-RAN侧激活端口并重新发起建链请求，观察告警是否清除。

      • 是 => 处理完毕。
      • 否 => 步骤10

    5. 与NG-RAN维护人员确认：NG-RAN侧是否存在导致端口或链路故障的异常告警。

      • 是 => 步骤9
      • 否 => 步骤10

    6. 由NG-RAN维护人员按照NG-RAN告警帮助排查NG-RAN故障，观察告警是否清除。

      • 是 => 处理完毕。
      • 否 => 步骤10

    7. 请联系华为技术支持。
