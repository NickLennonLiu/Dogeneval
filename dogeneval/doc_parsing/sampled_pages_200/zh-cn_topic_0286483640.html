QoS参数映射机制

为什么要进行QoS参数映射

在核心网内部的网元中存储着用户5QI、ARP、MBR/GBR等QoS信息，这些网元在进行消息转发时是知道哪些是优先转发，哪些是靠后转发的，但是消息离开这些网元进入IP承载网络时，路由器交换器等是不知道也看不到5QI、ARP等信息，为了和网元保持相同优先级转发报文，消息在离开网元进入这些路由器时需要进行QoS参数映射，这些路由器就可以依照映射后的QoS参数对不同报文进行不同优先级转发了。

为了保证无线用户的QoS要求能够传递到承载网，实现真正的E2E QoS，在UPF向DN侧转发用户报文时，需要将无线QoS参数值映射为承载网QoS参数值，也就是UPF需要在N6口发送数据包的时候，修改数据包的QoS标记，把核心网设备能理解的QoS转换为承载网设备能理解的QoS；在UPF向RAN侧转发用户报文时，也需要将承载网QoS参数值映射为无线QoS参数值，也就是UPF需要在N3口发送GTP包的时候，按照映射构造符合DSCP规划的GTP外层IP包，把承载网设备能理解的QoS转换为核心网设备能理解的QoS。

无线QoS与承载网QoS的映射过程

由于承载网中不同的网络使用不同的QoS优先级标记，例如VLAN网络使用802.1p，Diff-Serv网络使用DSCP（differentiated services code point区分服务编码点），MPLS网络使用EXP。核心网需要分别支持无线QoS到VLAN、Diff-Serv网络、MPLS网络的映射。详细的映射过程如图1所示。

图1 无线QoS与承载网QoS的映射过程
  • 上行数据报文映射过程
    1. 将上下文激活时确定的QoS参数以本地配置的规则，映射为DSCP或ToS值。
    2. 根据本地配置将DSCP映射为Service-Class和Color。
    3. 根据DN网络的类型，把Service-Class和Color映射为DSCP/EXP/802.1p的优先级。
    4. 将映射的规则填到数据包头进行转发，满足服务质量的要求。
  • 下行数据报文映射过程
    1. 根据DN网络类型，将DSCP/EXP/802.1p的优先级映射为Service-Class和Color值。
    2. 将Service-Class和Color值映射为DSCP值，将映射的DSCP值填入IP包头。
    3. 为了转发给RAN，在IP包的基础上封装IP+UDP+GTP-U包。
    4. 将上下文激活时确定的DSCP值，填到外层IP报头中。对于内层IP报头不做修改。该报文在核心网的传送过程中，将根据外层IP报头的DSCP或ToS获得不同的处理优先级，从而满足服务质量的要求。

无线QoS与承载网QoS的缺省映射关系

DiffServ是一种适用于骨干网络，可满足多种服务需求的IP QoS模型。在DiffServ体系中，用户通过标记分组的DS字段来申请不同层次的服务。DS字段的前6位称为DSCP，具有相同DSCP值的分组的集合称为BA（Behaviour Aggregate）。5QI到DSCP的默认映射关系如表1所示。如果5QI类型未在表中，请根据网络规划自行配置业务类型到DSCP的映射。其中，DSCP取值需经过端到端规划，满足传输设备中DSCP到PHB的映射规则。在合理规划的前提下，DSCP值越大则优先级越高。QoS与DSCP的映射规则可以通过ADD EPSREMARK和ADD 5GCREMARK命令调整。其中标准5QI等同于QCI，1：1映射，非标准的5QI根据SMF本地配置进行映射。

路由器保留有DSCP到每跳转发行为PHB（Per-Hop Behaviours）的映射，PHB是指满足某类转发需求的行为，如流量监管、流量整形、队列管理等。分组进入路由器时，根据其DSCP标记划归一个BA并被选取特定的PHB进行转发。核心网支持DSCP到PHB的映射，具体映射如表2所示，DSCP、802.1p与内部服务等级的映射规则可以通过SET QOSBA命令调整。

核心网网元根据用户报文的DSCP归划对应的具有相同DSCP值的分组集合BA并被选取特定的PHB进行转发。如果用户报文的DSCP未定义到PHB的映射，核心网网元将选择采用缺省PHB进行转发处理。目前，IETF定义了三种标准的PHB：

  • 加速转发EF（Expedited Forwarding）

    EF提供低丢包率、低延迟、高带宽的转发服务，它适用于信令的转发。

  • 确保转发AF（Assured Forwarding）

    AF提供对订购带宽质量的确保转发，超过订购带宽的转发具有不同丢弃优先级的服务，这主要适用于多媒体业务的转发。

  • 尽力而为BE（Best-Effort）

    BE即传统的IP分组投递服务，是缺省的PHB。只关注可达性，其他方面不做任何要求。适用于对带宽、延迟性能不敏感的E-Mail等业务。
