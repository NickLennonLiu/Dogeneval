操作步骤

  1. 进入“MML命令行-UNC”窗口。
  2. 打开本特性的License配置开关。

    SET LICENSESWITCH

  3. 配置SMF选择PCF的负荷分担参数。

    SET PCCFUNC

  4. 增加本地配置的对端NF实例的群组信息。

    ADD PNFGROUPID

  5. 指定服务发现策略。

    SET NFDISCPLCY

  6. 设置PCC故障处理动作。

    SET PCCFAILACTION

任务示例

任务描述

本实例中，操作员在SMF上进行数据配置实现以下要求：

  • PCF1~PCF3规划在一个PCF Group。

  • UNC获取PCF Group及其关联的PCF信息的方式为本地优先。

  • 当UE更新会话失败时，回滚为本地PCC用户，由备用PCF继续为用户服务。

脚本

//打开本特性的License配置开关。

SET LICENSESWITCH: LICITEM="LKV2NFSRA01",SWITCH=ENABLE;

//配置SMF选择PCF的负荷分担参数。

SET PCCFUNC: PCFLBPARA=GROUPID;

//本地配置对端NF实例的群组信息。

ADD PNFGROUPID: NFINSTANCEID="PCF_Instance_0", GROUPID="GROUP01";
ADD PNFGROUPID: NFINSTANCEID="PCF_Instance_1", GROUPID="GROUP01";
ADD PNFGROUPID: NFINSTANCEID="PCF_Instance_2", GROUPID="GROUP01";

//指定服务发现策略

SET NFDISCPLCY: POLICY=LOCAL_FIRST;

//设置PCC故障处理动作

SET PCCFAILACTION: UPDATEFAILACT=ROLLBACK, UPDFAILLOCPCC=LOCAL_PCC;
