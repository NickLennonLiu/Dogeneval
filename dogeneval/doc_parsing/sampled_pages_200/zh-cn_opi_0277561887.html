操作场景

当运营商需要部署GW-U故障隔离功能时，需要对本功能进行配置，确保本功能可以正常使用。

适用于SGW-C、PGW-C、GGSN。

必备事项

前提条件

  • 请仔细阅读WSFD-213004 GW-U故障隔离特性概述。
  • 完成加载License。

数据

S11、S5/Gn接口已在初始配置中完成，UPF的添加已在初始配置中完成。

操作步骤

  1. 进入“MML命令行-UNC”窗口。
  2. 打开本特性的License配置开关。

    SET LICENSESWITCH

  3. 配置SGW-U和S11的绑定关系。

    ADD UPBINDS11

  4. 配置GGSN-U/PGW-U和S5_P/Gn接口的绑定关系。

    ADD UPBINDGNGP

  5. 设置PGW-C发送给MME的失败原因值。

    ADDSPGWCAUSECTRL

任务示例

任务描述

任务1：配置SGW-U故障隔离功能。

任务2：配置GGSN-U/PGW-U故障隔离功能。

脚本

任务1：配置SGW-U故障隔离功能。

//打开本特性license。
SET LICENSESWITCH: LICITEM="LKV3GWUISL01", SWITCH=ENABLE;
//设置SGW-U和S11接口绑定信息。
ADD UPBINDS11: NFINSTANCENAME="UPF1",S11IPTYPE=Ipv4,S11IPV4 ="192.168.1.2",PRIORITY=3;
ADD UPBINDS11: NFINSTANCENAME="UPF2",S11IPTYPE=Ipv4,S11IPV4 ="192.168.1.2",PRIORITY=3;

任务2：配置GGSN-U/PGW-U故障隔离功能。

//打开本特性license。
SET LICENSESWITCH: LICITEM="LKV3GWUISL01", SWITCH=ENABLE;
//配置GGSN-U和Gn/Gp接口绑定信息。
ADD UPBINDGNGP: NFINSTANCENAME="UPF1",GNGPIPTYPE=Ipv4,GNGPIPV4="192.168.1.12";
ADD UPBINDGNGP: NFINSTANCENAME="UPF2",GNGPIPTYPE=Ipv4,GNGPIPV4="192.168.1.12";
//配置PGW-C发送给MME的失败原因值。
ADD SPGWCAUSECTRL: PROCEDURETYPE=CRT_SES_PROC, CAUSESOURCE=UPF, CAUSEEXPIRATION=73;
